## 1. 概要・背景

社員が業務上の困りごとや改善要望を登録し、担当部署が受付・対応・完了まで管理するアプリ。

例:
- 会議室の設備が壊れている
- 手作業のExcel集計を自動化してほしい
- 社内ルールが分かりにくい
- 備品を追加してほしい
- Power Automateで通知を自動化してほしい

といった依頼を一元管理する。

### 1.1 開発目的

現在、社内の改善要望や問い合わせは、Teams・メール・口頭など複数の方法で依頼されている。そのため次の問題がある。

- 誰が対応しているか分からない
- 同じ依頼が重複する
- 対応状況を依頼者が確認できない
- 過去の対応内容が蓄積されない
- 対応期限を忘れることがある

これらを解決するため、Power Appsで改善依頼管理アプリを作成する。

### 1.2 スコープに関する補足

- 「対応期限を忘れることがある」への対応は、初期リリースではホーム画面・ダッシュボードでの期限超過件数の可視化にとどめる。期限接近時の自動通知（Power Automate連携など）は行わない。要望があれば追加検討する。
- 「同じ依頼が重複する」への対応は、専用の重複チェック機構を設けず、②依頼一覧画面の検索・絞り込み機能で既存依頼を確認できることで対応する想定とする。

## 2. 利用者ロールと権限

ロールは排他的ではなく加算的。全ユーザーが「一般ユーザー」としての機能（自分の依頼の登録・確認）を持ち、その上に「担当者」「管理者」の権限が追加される。担当者・管理者も自分自身の依頼を登録・確認する場合は一般ユーザーとしての機能を使う。

### 一般ユーザー

- 改善依頼を登録する
- 自分が登録した依頼を確認する
- 依頼へコメントを追加する
- 対応完了後に満足度を入力する（「完了にする」操作時に入力）

### 担当者

- すべての依頼を確認する
- 自分を担当者として設定する（③.4節「受付する」ボタンの押下により、その場で自分が担当者として設定される。事前の自己アサイン操作は不要）
- ステータスを変更する
- 対応内容を記録する
- 期限を設定する

「担当部署」（⑦マスタ管理）は依頼の分類のための情報であり、誰が担当者になれるかを制限するものではない。部署に関係なく、担当者権限を持つ全ユーザーがどの依頼にも「受付する」操作で担当者になれる。

### 管理者

- 全依頼を確認・編集する
- 担当者を変更する
- カテゴリを管理する
- ダッシュボードを確認する
- 依頼を非表示または削除する

### 権限ルール

- 一般ユーザーは、自分が登録した依頼のみ編集可能。
- ただし「受付済み」以降は依頼内容を変更できない。
- 公開範囲が「関係者のみ」の依頼は、依頼者・担当者・関係部署所属ユーザー以外には一覧・詳細とも表示しない（[0章 #1](#0-レビューで確定した事項要件整理xlsxには無い追加仕様)）。

## 3. ステータスと遷移ルール

### 3.1 基本ステータス（正常フロー）

```
未受付 → 受付済み → 対応中 → 確認待ち → 完了
```

### 3.2 例外ステータス

- 差戻し
- 保留
- 対応不可
- 取消

### 3.3 ステータス遷移表

| 現在の状態 | 変更可能な状態 |
|---|---|
| 未受付 | 受付済み、取消 |
| 受付済み | 対応中、保留、対応不可 |
| 対応中 | 確認待ち、保留 |
| 確認待ち | 完了、差戻し |
| 差戻し | 対応中 |
| 保留 | 対応中、対応不可 |

### 3.4 ステータス別の操作ボタン（画面④の仕様に対応）

| 現在のステータス | 操作者 | 表示ボタン | 遷移先 | 備考 |
|---|---|---|---|---|
| 未受付 | 担当者 | 受付する | 受付済み | 押下した担当者がそのまま担当者として設定される |
| 未受付 | 依頼者 | 取消 | 取消 | |
| 受付済み | 担当者 | 対応開始 | 対応中 | |
| 受付済み | 担当者 | 保留 | 保留 | |
| 受付済み | 担当者 | 対応不可 | 対応不可 | |
| 対応中 | 担当者 | 確認依頼 | 確認待ち | |
| 対応中 | 担当者 | 保留 | 保留 | |
| 確認待ち | 依頼者 | 完了にする | 完了 | 押下時に満足度入力を挟む |
| 確認待ち | 依頼者 | 差し戻す | 差戻し | |
| 差戻し | 担当者 | 対応開始 | 対応中 | 受付済み時と同じボタンを再表示 |
| 保留 | 担当者 | 対応開始 | 対応中 | |
| 保留 | 担当者 | 対応不可 | 対応不可 | |

`完了`・`対応不可`・`取消` は終端ステータスであり、そこからの遷移はない。

## 4. 画面構成

### ① ホーム画面

表示内容（役割によって差がある）:

| 表示項目 | 一般ユーザー | 担当者 | 管理者 |
|---|---|---|---|
| 自分の未完了依頼件数 | ○ | ○ | ○ |
| 自分の完了依頼件数 | ○ | ○ | ○ |
| 担当中の依頼件数 | − | ○ | ○ |
| 期限超過件数 | 自分の依頼のうち期限超過 | 自分が担当中のうち期限超過 | 全社分と自分が担当中の分の両方を表示 |
| 新規依頼ボタン | ○ | ○ | ○ |
| 自分の依頼一覧 | ○ | ○ | ○ |
| 担当者向け依頼一覧への遷移ボタン | − | ○ | ○ |

- カード形式で件数を表示する。
- 「自分の依頼一覧」はプレビュー表示（直近5件程度）とし、全件は②依頼一覧画面へのリンクから確認する。
- 「担当者向け依頼一覧への遷移ボタン」は、新規の画面ではなく②依頼一覧画面へ「自分の依頼のみ」のチェックを外した状態（全件ビュー）で遷移するボタンとする。

### ② 依頼一覧画面

**表示項目**: 依頼番号、タイトル、カテゴリ、優先度、ステータス、依頼者、担当者、登録日、対応期限

**検索・絞り込み条件**: フリーワード、ステータス、カテゴリ、優先度、担当者、自分の依頼のみ、期限超過のみ

**並び替え**: 登録日の新しい順、対応期限の近い順、優先度順

**表示制御**: 公開範囲が「関係者のみ」の依頼は、依頼者・担当者・関係部署所属ユーザー以外の一覧には表示しない（一般ユーザー向け表示の場合）。担当者・管理者は全件閲覧可。

**レイアウト**: PCでは表形式、狭い画面ではカード形式に切り替える。（本アプリはPC専用のため表形式を主とする。[SCREEN_RENDERING_RULES.md](SCREEN_RENDERING_RULES.md) 参照）

**初期フィルタ状態**（画面を開いたときのデフォルト）:

| 遷移元 | 初期状態 |
|---|---|
| ①ホーム画面「自分の依頼一覧」（一般ユーザー） | 「自分の依頼のみ」ON |
| ①ホーム画面「担当者向け依頼一覧への遷移ボタン」 | 「自分の依頼のみ」OFF（全件） |
| 単独でこの画面を開いた場合（一般ユーザー） | 「自分の依頼のみ」ON |
| 単独でこの画面を開いた場合（担当者・管理者） | 「自分の依頼のみ」OFF（全件） |

**ページング**: 無限スクロールとする。[CODING_STANDARDS.md](CODING_STANDARDS.md) のデリゲーション対応（Data row limit 2000件）に達した場合は、`Notify(NotificationType.Warning)` 等で「これ以上は表示されません。絞り込み条件を追加してください」といった注意メッセージを表示する。

### ③ 依頼登録画面

**入力項目**:

| 項目 | 必須 | 内容 |
|---|---|---|
| タイトル | ○ | 50文字以内 |
| カテゴリ | ○ | マスタから選択 |
| 依頼内容 | ○ | 複数行テキスト |
| 希望対応日 | − | 日付 |
| 優先度 | ○ | 低・中・高・緊急 |
| 添付ファイル | − | 画像、Excel、PDFなど |
| 関係部署 | 条件付き必須 | 複数選択。⑦マスタ管理の「担当部署」と同一マスタを参照する。公開範囲＝「関係者のみ」を選択した場合は必須（[0章 #1](#0-レビューで確定した事項要件整理xlsxには無い追加仕様)の「関係者＝関係部署所属者」の判定に使うため） |
| 公開範囲 | ○ | 全社、関係者のみ |

- 依頼番号・依頼者・登録日・ステータス・担当者はシステム側で自動設定する（入力項目には含まない）。
- 登録時は確認ダイアログを表示する。「この内容で依頼を登録しますか？」［キャンセル］［登録する］

### ④ 依頼詳細画面

**表示内容**: 依頼情報、ステータス、担当者、対応期限、添付ファイル、対応内容、コメント履歴、ステータス変更履歴、満足度（ステータスが「完了」になった依頼のみ表示）

**編集権限**: 一般ユーザーは自分の依頼のみ編集可能。ただし「受付済み」以降は依頼内容を変更できない。

**対応期限の設定**: 担当者が、ステータスが「対応中」の間はいつでもこの画面から編集できる。

**操作ボタン**: [3.4 ステータス別の操作ボタン](#34-ステータス別の操作ボタン画面④の仕様に対応)を参照。

### ⑤ コメント画面・コメント領域

依頼者と担当者がコメントを追加できる。ステータスが「完了」「対応不可」「取消」などの終端状態になった後もコメントの追加を許容する（お礼のコメントなどを想定）。

**コメント項目**: コメント本文、投稿者、投稿日時、添付ファイル、社内担当者のみ表示フラグ

- 「社内担当者のみ表示」フラグを立てられるのは担当者・管理者のみ。依頼者はこのフラグを設定できない（自分のコメントを非公開にする意味がないため）。
- 「社内担当者のみ表示」にしたコメントは、一般ユーザーには表示されない。
- コメントの編集・削除機能は設けない（投稿は履歴として残り続ける）。要望があれば追加検討する。

### ⑥ ダッシュボード画面

対象: 管理者・担当者向け。表示される数値はすべて全社・全体集計であり、閲覧者個人によるフィルタはかからない（個人の数字は①ホーム画面が担う）。

**表示内容**:

- ステータス別件数
- カテゴリ別件数
- 月別登録件数
- 平均対応日数（= 完了日 − 登録日）
- 期限超過件数
- 担当者別の対応中件数
- 満足度平均

**期間フィルタ**: 「今月」「今四半期」「今年度」「任意期間指定」を設ける。月別登録件数・平均対応日数・期限超過件数など、期間に依存する集計はこのフィルタに従って再計算する。

Power Apps内では簡易チャートとし、将来的にはPower BIへの展開を想定する。

### ⑦ マスタ管理画面

**管理対象**: カテゴリ、担当部署、優先度、ステータス、アプリ管理者

- カテゴリ・担当部署・優先度は、管理者が自由に追加・編集・削除できる。
- ステータスのみ例外で、表示名・並び順など属性のみ編集可能。種類（9種類: 未受付・受付済み・対応中・確認待ち・完了・差戻し・保留・対応不可・取消）自体の追加・削除は不可（3章の遷移ルールに固定紐付けのため）。

**カテゴリ例**: システム、業務改善、設備、備品、人事・総務、ルール・手順、その他
